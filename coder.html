About & Disclaimer

Getting started — what you need

① A code editor. VS Code is the most common choice — download and install it like any app. Cursor is an AI-focused alternative that also works. All the extensions below install the same way in both.

② Ollama. This runs the AI model on your computer. Download and install it from ollama.com. Once installed, the ollama command is available in your terminal, and Ollama runs in the background automatically.

③ A coding assistant extension. Open VS Code, click the Extensions icon in the left sidebar (or press Ctrl+Shift+X / Cmd+Shift+X), search for the extension name, and click Install:

  • Cline — for agent models. Search "Cline". After installing, a Cline icon appears in the sidebar. Click it, then click ⚙️ to enter your settings.
  • Continue — for code chat and autocomplete models. Search "Continue". After installing, a Continue icon appears in the sidebar. Click it, then ⚙️ → Open Config to edit the config file.

Continue also works in Cursor, VSCodium, Windsurf, and JetBrains IDEs — the config is identical.

Data sources

Architecture parameters and capabilities are scraped from ollama.com. Run ollama show <model> to verify values for your installed variant.

Speed estimates are derived from GPU bandwidth and TFLOPS figures from public manufacturer data. Actual performance varies by driver, system load, and context length.

Coding rank

Models are ranked by a weighted score: 50% generation speed (most important in sequential agent loops), 30% context (more of the codebase visible at once), and 20% quality (quantization sharpness). This is a heuristic — not a benchmark.

AGENT vs CODE vs FIM

AGENT — purpose-trained for agentic coding loops (Devstral, Devstral Small). These models are fine-tuned on software engineering tasks: multi-step planning, file edits, shell commands, tool calling. Use with Cline or Continue in agentic mode. The top-ranked agent is marked ★ recommended — start here for vibe coding.

CODE — models that excel at code chat, explanation, review, and generation (CodeLlama, CodeGemma, Granite Code, Phi-4, Mistral Small). Use with Continue for IDE coding assistance. Some support tool-calling but are not optimized for autonomous coding agent loops — for that, choose an AGENT model. Cline is not shown for CODE models.

FIM — fill-in-the-middle models (Codestral, StarCoder2) for IDE cursor autocomplete. These use special FIM tokens — configure as tabAutocompleteModel in Continue. Not for chat or autonomous agent use.

Only curated coding models appear here. General chat models that merely support tool-calling (Llama, Mistral, Mixtral…) are excluded — they are not specifically optimized for coding.
