      These are estimates.
      Actual VRAM usage varies with ollama version, quantization, batch size, and OS/driver overhead —
      the % overhead reserve is a heuristic, not a guarantee.
      


      Color coding reflects how much of the model's architectural context limit fits in your VRAM:
      green ≥ 66%,
      amber ≥ 33%,
      orange < 33%.
      Orange means the model still runs — you just have a shorter context window than the model was designed for.
      


      This site's data was compiled by AI (Claude), not manually verified by a human.
      Model metadata (organization, origin, specialty) and HuggingFace config URLs were generated from AI training knowledge
      and may contain errors. Architecture parameters are cross-checked against HuggingFace config.json
      via an automated script, but the underlying URLs were AI-generated and not all have been confirmed live.
      


      Weight sizes are sourced from the ollama registry manifest (exact layer bytes, no model downloads).
      Quantization varies by tag — run ollama show <model> for exact values.
      Architecture parameters (layers, heads, head_dim, max_position_embeddings) are sourced from
      HuggingFace config.json — verify for fine-tuned variants.
      KV cache assumes fp16 (ollama default); some builds use fp32, which doubles the cache size.
      Always monitor nvidia-smi when pushing context limits.
