      These are estimates.
      Actual VRAM usage varies with ollama version, quantization, batch size, and OS/driver overhead —
      the % overhead reserve is a heuristic, not a guarantee.
      


      Color coding reflects how much of the model's architectural context limit fits in your VRAM:
      green ≥ 66%,
      amber ≥ 33%,
      orange < 33%.
      Orange means the model still runs — you just have a shorter context window than the model was designed for.
      


      This site's data was compiled by AI (Claude), not manually verified by a human.
      Model metadata (organization, origin) was inferred from AI training knowledge and may contain errors.
      


      Weight sizes and architecture parameters are sourced entirely from ollama.com —
      the model detail page (weights, parameter count) and the model blob page (layers, heads, head_dim,
      max_position_embeddings). Quantization varies by tag — run ollama show <model>
      for exact values. Verify for fine-tuned variants.
      KV cache defaults to f16. Set OLLAMA_KV_CACHE_TYPE=q8_0 or q4_0
      to halve or quarter the cache size — requires Flash Attention.
      Flash Attention is opt-in in ollama because it is not universally faster: benefits scale with context length
      (negligible below 4k, meaningful above 8k), and newer GPU generations (H100, A100, RTX 40xx) gain more than older ones.
      On AMD, Apple Silicon, and Intel Arc support is limited or experimental.
      Always monitor nvidia-smi when pushing context limits.
